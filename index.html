🌐 인터넷과 웹의 기본 개념

코딩을 배우기 전에, 인터넷이 어떻게 돌아가는지부터 이해해요.

🖥️ 클라이언트 vs 서버

☕ 카페 비유

손님(클라이언트)이 직원(서버)에게 "아메리카노 주세요"라고 요청하면, 직원이 커피를 만들어 건네줍니다.
→ 우리가 주소창에 주소를 입력하면(요청, Request), 서버가 페이지를 보내줍니다(응답, Response).

👤 클라이언트 (Client)

우리 컴퓨터·핸드폰
웹브라우저(Chrome, Edge)가 역할
요청을 보내는 쪽

🖥️ 서버 (Server)

데이터를 저장하고 처리
항상 켜져 있는 컴퓨터
요청에 응답하는 쪽

브라우저가 하는 일: ① 사용자가 URL 입력 → ② 서버에 요청 → ③ HTML 파일 받음 → ④ 화면에 표시

🔗 URL 완전 분석

URL(Uniform Resource Locator)은 인터넷 주소예요. 각 부분이 다 의미가 있어요.

https://fieldtrip-hyehwa.vercel.app/teacher/scope?year=2026 │ │ │ │ │ │ 프로토콜 서브도메인 도메인 경로1 경로2 쿼리파라미터
https://
프로토콜 — 암호화된 안전한 통신 방식 (http는 암호화 없음)
.vercel.app
도메인 — 서버를 찾는 주소 (전화번호 같은 것)
/teacher/scope
경로(Path) — 서버 안에서 어느 페이지인지
?year=2026
쿼리 파라미터 — 추가 데이터 전달 (검색어, 필터 등)

🏗️ 프론트엔드 vs 백엔드

🎨 프론트엔드 (Front-end)

눈에 보이는 부분
버튼, 색상, 레이아웃
HTML · CSS · JavaScript

브라우저에서 실행됨

⚙️ 백엔드 (Back-end)

눈에 안 보이는 부분
데이터 저장·조회, AI 호출
Node.js · Python · DB

서버에서 실행됨

내가 만든 세특 사이트: 버튼·입력폼은 프론트엔드 / AI 호출·DB 저장은 백엔드(Vercel 서버)

📡 HTTP 요청의 종류

브라우저와 서버가 대화할 때 쓰는 "동사"예요. 각각 다른 의미를 가져요.

GET
데이터 가져오기. 학생 목록 조회, 페이지 열기 (가장 흔함)
POST
데이터 보내기/생성. 회원가입, 세특 저장, AI 요청
PUT/PATCH
데이터 수정. 세특 내용 업데이트
DELETE
데이터 삭제. 학생 정보 삭제
URL 주소창에서 Enter 치는 건 항상 GET 요청이에요. POST는 보통 폼 제출이나 버튼 클릭으로 발생해요.

📊 HTTP 상태 코드

서버가 응답할 때 숫자로 결과를 알려줘요. 내가 코딩할 때 console에서 자주 보여요.

200 OK
성공! 요청한 데이터를 잘 받았어요
201 Created
생성 성공! 새 데이터가 만들어졌어요
400 Bad Request
요청 형식이 잘못됨. 내가 보낸 데이터에 문제가 있어요
401 Unauthorized
로그인이 필요해요. 인증되지 않은 접근
403 Forbidden
권한 없음. 로그인은 됐지만 접근 불가
404 Not Found
페이지/데이터가 없어요. 주소가 틀렸거나 삭제됨
500 Server Error
서버에서 오류 발생. 내 코드(서버 측)에 버그가 있어요

웹 페이지의 뼈대를 만드는 언어예요. 태그로 구조를 만들어요.

📄 HTML 문서 기본 구조

모든 HTML 파일은 이 구조로 시작해요. 외울 필요 없이 이해만 하면 돼요.

📝 텍스트 태그

<h1> ~ <h6>
제목 (h1이 가장 크고 h6이 가장 작음). 한 페이지에 h1은 1개만!
<p>
문단(Paragraph). 일반 텍스트 내용. 위아래 여백 자동 생김
<strong>
굵은 글씨. 중요한 단어 강조할 때
<em>
기울어진 글씨 (이탤릭체). 강조나 특별한 의미
<span>
인라인 컨테이너. 텍스트 일부에만 스타일 줄 때
<br>
줄바꿈 (닫는 태그 없음). Enter와 같은 효과
<hr>
수평 구분선. 내용 구분할 때
<code>
코드 형식으로 표시. 글씨체가 고정폭으로 바뀜

🏛️ 시맨틱(Semantic) 태그 — 의미 있는 구조

예전엔 모든 걸 <div>로 했지만, 이제는 의미를 담은 태그를 써요. 검색엔진이 이해하기 쉽고, 코드가 더 읽기 좋아요.

<header>사이트 상단 헤더 (로고, 메뉴)</header> <nav>네비게이션 메뉴</nav> <main> <section>주제별 구역 (학생 목록 섹션 등)</section> <article>독립적인 콘텐츠 (공지사항 하나)</article> <aside>사이드바, 관련 정보</aside> </main> <footer>하단 푸터 (저작권, 연락처)</footer>
div는 "그냥 상자"예요. 위의 태그들은 "이게 뭔지" 알려주는 상자예요. 내가 코딩할 때 이 태그들이 보이면 레이아웃 구조를 파악하는 데 쓰여요.

📊 테이블 태그

표를 만들 때 써요. 내가 성적표나 학생 목록 만들 때 자주 쓰는 구조예요.

<table> <thead> <!-- 표 헤더 영역 --> <tr> <!-- tr = 행(row) --> <th>이름</th> <!-- th = 헤더 셀 (굵게 표시됨) --> <th>학번</th> </tr> </thead> <tbody> <!-- 표 내용 영역 --> <tr> <td>홍길동</td> <!-- td = 일반 셀 --> <td>10101</td> </tr> </tbody> </table>

📝 폼(Form) 태그 — 데이터 입력

사용자에게 입력을 받을 때 써요. 로그인, 검색, 설문조사 등 모두 폼이에요.

🎨 CSS 완전 정복

화면을 예쁘게 꾸미는 언어예요. 선택자·박스모델·레이아웃 순서로 익혀요.

🎯 CSS 셀렉터 (어느 요소에 스타일을 줄까?)

CSS에서 가장 중요한 개념이에요. "어떤 태그/클래스를 선택하느냐"가 핵심이에요.

/* 태그 셀렉터 — h1 태그 전부 */ h1 { color: purple; } /* 클래스 셀렉터 — class="card" 인 요소들 */ .card { background: white; border-radius: 12px; } /* ID 셀렉터 — id="title" 인 딱 하나 */ #title { font-size: 2rem; } /* 자손 셀렉터 — .container 안에 있는 p 태그 */ .container p { color: gray; } /* 가상 클래스 — 마우스 올렸을 때 */ button:hover { background: #9370db; } /* 가상 클래스 — 첫 번째 자식 */ li:first-child { font-weight: bold; }

📦 박스 모델 (Box Model) — CSS의 핵심 중 핵심

🎁 택배 상자 비유

내용물(content) → 뽁뽁이(padding) → 상자 벽(border) → 상자들 사이 간격(margin)
모든 HTML 요소는 이 4겹 구조예요.

/* 박스 모델의 4가지 영역 */ .box { width: 200px; /* 내용 너비 */ height: 100px; /* 내용 높이 */ padding: 16px; /* 안쪽 여백 (내용↔테두리 사이) */ padding: 10px 20px; /* 위아래 10px, 좌우 20px */ border: 2px solid #ccc; /* 테두리 (두께 스타일 색) */ border-radius: 12px; /* 모서리 둥글게 */ margin: 20px; /* 바깥쪽 여백 (다른 요소와의 간격) */ margin: 0 auto; /* 가운데 정렬 (좌우 auto) */ box-sizing: border-box; /* padding/border를 width에 포함 (권장) */ }

📐 Display 속성 — 요소가 어떻게 배치될까?

block
한 줄 전체를 차지. 다음 요소는 아랫줄로. div, p, h1이 기본값
inline
내용 크기만큼만 차지. 옆으로 나란히 배치. span, a가 기본값. width/height 설정 안 됨
inline-block
옆으로 나란히 + width/height 설정 가능. 버튼처럼
flex
자식들을 가로/세로로 유연하게 배치. 현대 레이아웃의 핵심!
grid
격자(바둑판) 형태로 배치. 복잡한 레이아웃에 강력
none
완전히 숨김. 공간도 차지 안 함. 탭 전환 등에 사용

🔧 Flexbox — 현대 레이아웃의 핵심

내가 만드는 모든 레이아웃의 70%는 Flexbox예요. 이것만 잘 이해해도 레이아웃은 OK!

/* 부모 컨테이너에 flex 설정 */ .container { display: flex; /* 방향 */ flex-direction: row; /* 가로 (기본값) */ flex-direction: column; /* 세로 */ /* 주축 정렬 (가로방향일 때 좌우) */ justify-content: flex-start; /* 왼쪽 정렬 */ justify-content: center; /* 가운데 */ justify-content: flex-end; /* 오른쪽 */ justify-content: space-between;/* 양 끝에 배치, 사이 간격 균등 */ /* 교차축 정렬 (가로방향일 때 위아래) */ align-items: center; /* 세로 가운데 정렬 */ align-items: stretch; /* 높이 맞춤 (기본값) */ gap: 16px; /* 자식들 사이 간격 */ } /* 자식 요소 */ .item { flex: 1; /* 남은 공간 균등 분배 */ flex: 0 0 200px; /* 200px 고정 */ }

📍 Position 속성

static
기본값. 일반 흐름대로 배치
relative
자기 원래 위치 기준으로 이동. top/left로 조절
absolute
부모 기준으로 위치 고정. 일반 흐름에서 빠져나옴
fixed
브라우저 화면 기준 고정. 스크롤해도 그 자리에 (상단 nav 등)
sticky
스크롤하다가 특정 지점에서 fixed처럼 고정됨
/* 예시: 오른쪽 상단에 고정된 배지 */ .parent { position: relative; } .badge { position: absolute; top: 8px; right: 8px; }

📱 반응형 디자인 (Media Query)

화면 크기에 따라 다른 스타일을 적용해요. 핸드폰에서도 잘 보이게 하는 핵심이에요.

/* 기본: 모바일 우선 설계 */ .card { width: 100%; } /* 화면이 768px 이상일 때 (태블릿/PC) */ @media (min-width: 768px) { .card { width: 48%; } } /* 화면이 1024px 이상일 때 (PC) */ @media (min-width: 1024px) { .card { width: 31%; } }

🔄 CSS 변수 (Custom Properties)

자주 쓰는 색상이나 크기를 변수로 만들어 관리해요. 내가 코딩할 때 거의 항상 쓰는 패턴이에요.

📦 JavaScript — 변수와 데이터 타입

프로그래밍의 가장 기초. 데이터를 담는 그릇(변수)과 데이터의 종류(타입)부터 시작해요.

🗃️ 변수 선언 — var / let / const

📦 상자 비유

변수는 데이터를 넣어두는 상자예요. 상자 이름을 지어주고, 안에 값을 넣는 거예요.
const = 잠금 상자 (바꾸면 안 됨) | let = 일반 상자 (바꿀 수 있음)

// const — 바꾸지 않을 값 (권장 기본값) const schoolName = "혜화여자고등학교"; const studentCount = 30; // let — 바뀔 수 있는 값 let score = 85; score = 90; // 나중에 변경 가능 // var — 옛날 방식. 지금은 잘 안 씀 (버그 생기기 쉬움) var oldStyle = "사용 비권장";
const를 기본으로 쓰고, 값이 바뀌어야 할 때만 let을 써요. var는 이제 안 써요!

🔢 데이터 타입 7가지

🔷 컴포넌트 (Component)

React의 기본 단위예요. 함수가 JSX(HTML처럼 생긴 것)를 반환하는 형태예요.

📸 사진 앨범 비유

코드를 수정할 때마다 스냅샷(commit)을 찍어요. 나중에 이전 상태로 언제든 되돌아갈 수 있어요.
GitHub는 그 사진 앨범을 클라우드에 저장하는 곳이에요.

# 처음 시작 git init # Git 시작 git remote add origin [깃허브URL] # 원격 저장소 연결 # 매일 쓰는 기본 흐름 git status # 변경된 파일 확인 git add . # 모든 변경 파일 스테이징 git add 파일명 # 특정 파일만 스테이징 git commit -m "내용 수정" # 스냅샷 저장 (메시지 필수!) git push # GitHub에 업로드 → Vercel 자동 배포! git pull # GitHub에서 최신 코드 가져오기 # 기록 보기 git log --oneline # 커밋 목록 요약 git diff # 변경 내용 비교

🃏 컴포넌트 — UI 조각들

카드 Card

📍 어디서 봄? → 쇼핑몰 상품 목록, 인스타그램 게시물, 학생 정보 카드

버튼 Button
클릭해서 동작을 실행하는 요소예요. 목적에 따라 여러 종류가 있어요. 가장 중요한 동작은 Primary(보라/파랑), 취소는 Secondary, 위험한 동작은 Danger(빨강)로 구분해요.
📍 Primary = 메인 동작 | Secondary = 취소/보조 | Danger = 삭제 | Disabled = 비활성화

기본 버튼 종류

탭 Tab
클릭하면 아래 내용이 바뀌는 메뉴 버튼이에요. 브라우저의 탭과 같은 원리예요. 선택된 탭은 강조 표시(보통 색이나 밑줄)되고, 나머지는 흐릿하게 표시돼요.
📍 어디서 봄? → "정보" "사진" "동영상" 탭, 설정 페이지의 카테고리 탭

참여도, 점수 분포 등의 통계가 여기 표시됩니다.
모달 Modal / Dialog
배경을 어둡게 흐리고 위에 뜨는 팝업 창이에요. 사용자가 중요한 결정을 해야 할 때 사용해요. 모달이 열려있는 동안은 배경의 다른 요소를 클릭할 수 없어요.
📍 "정말 삭제하시겠습니까?" "로그인 해주세요" "사진 크게 보기" 등
예시

📍 어디서 봄? → "계열 선택", "학년 선택", "언어 설정"

📝 입력 폼 — Form Elements

인풋 Input Field
한 줄 텍스트를 입력하는 칸이에요. 이름, 학번, 비밀번호 입력에 사용해요. 상태마다 다르게 보여요: 기본/포커스(선택됨)/에러.
📍 placeholder = 입력 전 안내 문구 | label = 입력칸 설명
예시
기본 상태
포커스 상태 (클릭했을 때)
에러 상태
⚠ 학번 형식이 올바르지 않아요
텍스트에어리어 Textarea
여러 줄의 텍스트를 입력할 수 있는 큰 입력칸이에요. 세특 내용, 자기소개서, 게시글 작성 등 긴 글을 입력할 때 사용해요. 오른쪽 아래 모서리를 드래그해서 크기를 조절할 수 있어요.
📍 input(한 줄) vs textarea(여러 줄)
예시
세특 내용 작성
215자 / 500자
체크박스 Checkbox
여러 개 중에서 복수로 선택할 수 있는 네모 체크 버튼이에요. 체크(✓)된 것과 안 된 것을 함께 가질 수 있어요.
📍 체크박스 = 여러 개 동시 선택 가능 | 라디오 버튼 = 하나만 선택

🧭 네비게이션 — Navigation

네비게이션 바 Navbar
페이지 이동을 위한 메뉴 모음이에요. 보통 헤더 안에 있어요. 현재 페이지의 메뉴는 색이나 굵기로 강조 표시돼요. 모바일에서는 햄버거 메뉴로 숨겨지기도 해요.
📍 어디서 봄? → 거의 모든 웹사이트 상단 메뉴

← '교사 메뉴'가 현재 페이지라 강조 표시돼요
브레드크럼 Breadcrumb
지금 내가 어디 있는지 경로를 보여주는 내비게이션이에요. "홈 > 교사 메뉴 > 세특 관리" 처럼 표시해요. 헨젤과 그레텔의 빵가루(breadcrumb)처럼 길을 안내한다고 해서 붙여진 이름이에요.
📍 어디서 봄? → 쇼핑몰 "홈 > 여성의류 > 원피스" 같은 경로 표시
나의사이트.com/teacher/scope
🏠 홈 › 교사 메뉴 › 세특 관리 › 세특 생성
↑ 현재 위치: 세특 생성 페이지
페이지네이션 Pagination
긴 목록을 페이지로 나눠서 보여주는 UI예요. 1, 2, 3 페이지 번호가 있고, 이전/다음 버튼이 있어요. 한 번에 너무 많은 데이터를 보여주면 느려지기 때문에 사용해요.
📍 어디서 봄? → 학교 공지사항 목록, 쇼핑몰 상품 목록
